# 專案起源與對話摘要 (Project Genesis)

## 1. 核心願景

本專案旨在將頂級投資銀行（如 Goldman Sachs, Morgan Stanley）的分析邏輯自動化。透過 AI Agent 技術，從海量的市場數據中過濾雜訊，精準識別被市場低估的「隱藏價值股」與追蹤「熱門領導股」，並產出具備專業深度與操作建議的投資報告。

## 2. 演進歷程

### 第一階段：專業指令集定義

* **起始點**：整理並翻譯了 12 組投行核心分析模型（包括 DCF, LBO, M&A, SOTP, 單位經濟等）。
* **優化**：在基礎估值模型上，加入了「目標價預測」與「安全邊際（MOS）」概念，並整合了 **ICT (Inner Circle Trader)** 的技術面分析邏輯（如 FVG, Order Block）。

### 第二階段：數據驅動思維

* **數據拆解**：明確化每個分析模型所需的 Input 數據（如 WACC 參數、三表數據、同業乘數）。
* **獲取策略**：確定了以 `yfinance` 為主，結合 `Polygon.io` 與 `SEC EDGAR` 的多維度數據獲取路徑。

### 第三階段：架構自動化（漏斗模型）

* **挑戰**：解決 2000 檔股票全量分析的成本與精確度問題。
* **解決方案**：設計了**三層過濾漏斗**：
1. **量化層**：Python 硬指標篩選（價值/成長指標）。
2. **情緒層**：區分高討論度（Mag7）與低討論度（Hidden Gems）。
3. **分析層**：AI 扮演高階分析師進行深度推理。



### 第四階段：技術棧確定 (2026 Tech Stack)

* 採用 **Claude Skills + MCP (Model Context Protocol)** 模式。
* 利用 **Claude Code** 在本地進行開發與執行。
* 使用 **Python (uv)** 負責數據運算，**Svelte 5** 負責報告展示，**Telegram Bot** 負責實時推送。

## 3. 關鍵決策紀錄 (Key Decisions)

* **Decision 1**: 拒絕理想化的「2000 檔全量 AI 分析」，改採「Python 篩選 + AI 深度分析」的混合模式。
* **Decision 2**: 引入 MCP 協議，讓 AI 具備主動抓取本地暫存與在線數據的能力，而非被動接受餵食。
* **Decision 3**: 報告產出必須包含明確的「買賣建議、目標價與風險比」，提升實戰參考價值。
